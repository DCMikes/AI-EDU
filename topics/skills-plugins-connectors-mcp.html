Skills, Plugins, Connectors, Extensions, and MCP Servers: How They Fit Together

Written July 17, 2026 Updated July 17, 2026 v1.1

Claude's add-ons come with five names that sound interchangeable and are not: skills, plugins, connectors, extensions, and MCP servers. This page gives you a one-line definition of each, shows how they stack together, and points each one at real examples we already use at DCN.

Why

Why you would use this

The shortest way to keep them straight is to split them by what they add. Two of these give Claude knowledge: how to do a job, what steps to follow, how we word things. The other three give Claude reach: access to systems it could not touch on its own, like your email or LabTrac. Once you know which kind of problem you have, you know which piece you need, and you can ask IT for the right thing instead of "some AI thing that does X."

Here is each one in a line:

  1. Skill (knowledge): a folder of saved instructions, plus any templates or reference files, that teaches Claude how to do a specific job your way. Ours: brand-dcn, which carries our voice rules and brand assets. Covered in depth on the skills training page.
  2. MCP server (reach): a program that speaks the Model Context Protocol, an open standard that lets Claude safely use outside tools and data. It is the plumbing underneath everything in the "reach" family. Ours: FETCH, the read-only service that lets Claude answer questions from LabTrac case data.
  3. Connector (reach): the friendly, packaged form of an MCP server inside the Claude apps. You turn one on from a settings menu, sign in, and Claude gains those tools; no technical setup. Ours: the Microsoft 365 connector for Outlook, SharePoint, OneDrive, and Teams.
  4. Extension (reach): a one-click install file that packages an MCP server to run on your own computer, used by Claude Desktop. Same idea as a connector, but the tools run locally on your machine instead of in the cloud. The Claude for Chrome browser extension is a cousin: it gives Claude reach into your browser.
  5. Plugin (packaging): a bundle used to distribute the other pieces to a whole team at once. One plugin can carry skills, slash commands, and MCP server connections together. Ours: the brand-dcn plugin, which is how the brand-dcn skill reaches every DCN machine automatically.

How

How they interconnect

These are not five rival options; they stack. The interconnections are the part worth remembering:

  1. A skill can be part of a plugin. The skill is the content; the plugin is the box it ships in. brand-dcn is exactly this: you experience it as a skill, and IT delivers it as a plugin through our internal catalog.
  2. Connectors and extensions are both MCP servers in nicer clothes. A connector is an MCP server hosted in the cloud and switched on from settings; an extension is an MCP server installed on your own machine. Under the hood, the Microsoft 365 connector and our FETCH service speak the same protocol.
  3. A plugin can also carry MCP connections. Besides skills, a plugin can include the configuration that points Claude at an MCP server, so one install gives a teammate both the know-how and the access.
  4. Skills and MCP servers are better together. The MCP server provides the tools; a skill teaches Claude how to use them well. Our FETCH setup works this way: the server exposes LabTrac data, and a companion skill tells Claude which questions map to which queries and what the numbers mean.
  5. What stacks on what: MCP servers are the foundation of reach; connectors and extensions are two ways to plug them in; skills sit on top adding judgment and process; and plugins wrap any of it for team-wide delivery.

A practical rule of thumb: if Claude keeps doing a task the wrong way, you want a skill. If Claude cannot see the data at all, you want a connector, an extension, or an MCP server. If the whole team needs either one, ask IT about a plugin.

Prompts

Three prompts to try

  1. "What skills, connectors, and MCP tools do you have available in this chat right now? Group them by which system they reach."
  2. "I want you to pull numbers from LabTrac and format them the way our morning huddle expects. Which parts of that need a connector and which parts need a skill?"
  3. "Explain what the brand-dcn plugin contains and what would be different if it were only a skill on my machine."

One boundary worth knowing: connectors, extensions, and MCP servers grant real access to real systems, so at DCN those are set up or approved by IT. Skills are safe to experiment with on your own, and the skills page shows you how.
