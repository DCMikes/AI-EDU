Claude Skills: What They Are and How to Make One

Written July 17, 2026 Updated July 17, 2026 v1.2

A skill is a set of saved instructions, and often some bundled files, that teaches Claude how to do a specific job your way. Instead of re-explaining the same process every time, you write it down once and Claude loads it whenever the job comes up. This page explains what a skill is, how to use the ones we already have, and how to build your own.

Why

Why you would use this

Think about a task you explain the same way every time: how a case note should be worded, what goes in a weekly report, which template a one-pager starts from. Without a skill, that knowledge lives in your head, and every chat starts from zero. With a skill, the process is written down once, and every person on the team gets the same consistent result from a single short prompt.

You already use one every week: brand-dcn is a skill. It bundles our voice rules, colors, logos, and templates, so "write this email" comes out sounding like DCN without you pasting in the brand guide. Any repeatable process at DCN can work the same way: your process, saved once, applied every time.

Under the hood a skill is simple: a folder holding a file of plain language instructions (called SKILL.md), plus any reference files the job needs, such as templates, examples, or checklists. There is no programming involved. If you can write the steps in an email, you can write a skill.

How

How you use one and how you make one

Using a skill

Skills activate two ways. You can call one directly by typing a slash and its name, such as /brand-dcn, at the start of your request; that is the surest way to point Claude at it. Or you can just ask in plain language, and Claude activates the matching skill on its own when your request fits the skill's description. To see what you have, ask Claude "what skills do you have available?"

Making your own

  1. Pick one repeatable job with a clear right way to do it. Good first skills are narrow: "format my morning production summary," not "help me with everything."
  2. Ask Claude to build it with you: "Create a skill called case-followup that drafts follow-up notes for doctors using the steps I give you." Claude has a built-in skill-creator that asks what the job is, writes the SKILL.md file, and saves the folder in the right place.
  3. Tell it the process like you would train a new teammate: the steps, the wording rules, what good output looks like, and what to avoid. Attach an example or template if you have one; it gets bundled into the skill folder.
  4. Test it in a fresh chat by typing / plus its name with a real task. If something comes out wrong, say so and ask Claude to update the skill; the fix sticks for next time.
  5. If the skill would help the whole team, tell IT. We can publish it through our internal catalog the same way brand-dcn reaches everyone automatically.

Where skills live

A personal skill sits on your machine and only you see it. A shared skill can be packaged into a plugin and published so the whole team gets it, which is exactly how brand-dcn works today. The page on skills, plugins, connectors, extensions, and MCP servers covers how the pieces fit together.

Prompts

Three prompts to try

  1. "What skills do you have available right now, and what does each one do?"
  2. "Create a skill called meeting-recap that turns my rough notes into a short recap email with action items. I will give you my formatting rules."
  3. "Update my meeting-recap skill: action items should always include a name and a due date."

One boundary worth knowing: a skill is instructions, not a connection. It cannot reach systems Claude could not already reach, so it will not pull live data from LabTrac or your email on its own. For that, skills work alongside connectors, which are covered in their own training.
