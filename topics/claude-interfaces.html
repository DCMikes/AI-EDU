Ways to Use Claude: Chat, Cowork, Code, the CLI, and Office Plugins

Written July 17, 2026 Updated July 17, 2026 v1.0

It is the same Claude behind every one of these; what changes is the doorway you walk through and what Claude can reach once you are inside. This page walks through the five ways you are likely to meet Claude at DCN, why you would pick each one, and where each one stops.

Why

Why the doorway matters

People often decide Claude "cannot do" something when the real answer is that they were standing in the wrong doorway. Chat is perfect for a question and useless for editing the files in a folder. Cowork is built for producing a finished document and is more than you need for a quick rewrite. Picking the right surface for the job saves you the frustration of fighting a tool that was never meant for that task.

A helpful way to sort them: Chat and the Office plugins keep you in a conversation or a single document. Cowork, Code, and the Code CLI let Claude work across real files and projects and hand back finished work. All of them can use the same skills and connectors we set up, though how much of that they carry varies, as noted below.

How

The five doorways, and where each one stops

1. Claude Chat (web, desktop, and mobile app)

This is the everyday conversational Claude: claude.ai in a browser, the desktop app, and the phone app.

  • Why you would use it: it is the fastest way in and needs no setup beyond signing in. Ask questions, draft and review writing, work through a problem, and analyze documents you paste or upload. It runs skills such as brand-dcn and uses connectors such as the Microsoft 365 connector to pull your own email, files, and chats.
  • Where it stops: Chat works with what you give it or connect to. It does not reach into the files on your computer to edit them in place, and a chat is a conversation rather than a saved workspace. For work that spans many files or produces a polished document, reach for Cowork or Code instead. Uploads also have size limits.

2. Cowork

A shared workspace where Claude works alongside you on real deliverables, such as Word documents, spreadsheets, slide decks, and PDFs, using skills and plugins and handing back the finished files.

  • Why you would use it: when the thing you need is a file, not just an answer. A proposal, a formatted report, a deck built from our template. Claude creates and edits the actual document and iterates with you until it is right.
  • Where it stops: Cowork works against a working folder, and your changes only sync and become visible to teammates when that folder is a cloud location such as OneDrive or SharePoint. If you point it at files that live only on your machine, the work stays on your machine. Put anything you want to keep or share in a synced cloud folder before you start.

3. Claude Code

The tool for working inside codebases and technical projects, available in the desktop app, as an editor extension for VS Code and JetBrains, and on the web.

  • Why you would use it: when you need Claude to read across many files, make coordinated edits, run commands, and check the results. Building or maintaining software, working in a git repository, or automating file-heavy technical tasks. This very training site is maintained with Claude Code.
  • Where it stops: it is aimed at technical work and is more than you need for a quick question or a one-off email. Because it acts on real files and can run commands, it belongs with people who are comfortable reviewing what it changes before they accept it, and it needs the project available to it.

4. Claude Code CLI

The same Claude Code engine, run from your terminal (the command line) instead of a window.

  • Why you would use it: for people who already work in the terminal. It fits alongside existing scripts and commands, works over remote connections to other machines, and can be automated or scheduled to run on its own. Same capability as Code, driven by the keyboard.
  • Where it stops: it assumes you are comfortable at the command line and gives you no graphical file browser. If the terminal is not already part of your day, the desktop or editor version of Code is the friendlier door to the same power.

5. Plugins for Office

Claude brought inside the Microsoft Office apps: Word, Excel, PowerPoint, and Outlook, through an add-in, so you can ask Claude for help without leaving the document you are in.

  • Why you would use it: it keeps you in the tool you are already using. Draft a letter in Word, summarize a long thread in Outlook, or make sense of a sheet in Excel, all in place, without copying text back and forth to another window.
  • Where it stops: it is the newest and narrowest of the five, scoped to the document or message you are working in rather than your whole workspace, and it does not carry the full set of skills and connectors that a Chat or Cowork session does. Availability and install are handled by IT, so check with us before you count on it for a task.

Prompts

How to decide, and three prompts to try

A quick rule of thumb: reach for Chat for a question or a draft, Cowork when you need a finished file to keep or share, Code or the CLI for anything that lives in a code project, and the Office plugins when you would rather not leave Word, Excel, PowerPoint, or Outlook.

  1. "I need a branded one-pager I can save to our SharePoint and hand to a doctor. Which of these should I use, and why?"
  2. "In Cowork, start a proposal in my OneDrive folder so it syncs; use our letterhead template."
  3. "I am in Outlook and this thread is long. Summarize it and draft a reply in our voice."

One boundary worth repeating: these are doorways to the same Claude, but they are not equal in reach. If a task needs your live company data or a finished, shareable file, the doorway you choose decides whether that is even possible, so it is worth a moment's thought before you start.
